<objective>
## Goal
Create migration 010 that renames `documents` → `knowledge_sources` (source tracking), drops `document_chunks`, creates a new LangChain-compatible `documents` table (content/metadata/embedding), and replaces `match_document_chunks` with a `match_documents` RPC matching the LangChain SupabaseVectorStore signature.

## Purpose
LangChain's `SupabaseVectorStore` writes to a `documents` table (content text, metadata jsonb, embedding vector) and calls `match_documents(query_embedding, filter)`. The v1.0 schema uses the same table name for source tracking, which conflicts. This migration resolves the naming conflict and establishes the correct LangChain-compatible schema before the pipeline is wired.

## Output
- `supabase/migrations/010_knowledge_langchain.sql` created and pushed
- `knowledge_sources` table exists (source tracking, renamed from `documents`)
- `documents` table exists (LangChain vector store: content, metadata jsonb, embedding vector(1536))
- `match_documents` function exists with LangChain signature
- `document_chunks` and `match_document_chunks` removed
- RLS active on both tables
</objective>

<task type="auto">
  <name>Task 1: Write migration 010_knowledge_langchain.sql</name>
  <files>supabase/migrations/010_knowledge_langchain.sql</files>
  <action>
    Create the migration file with these steps in order:

    -- Step 1: Drop old function and dependent table
    DROP FUNCTION IF EXISTS public.match_document_chunks(uuid, extensions.halfvec, int, float);
    DROP TABLE IF EXISTS public.document_chunks CASCADE;

    -- Step 2: Rename source tracking table
    ALTER TABLE public.documents RENAME TO knowledge_sources;

    -- Rename the policy to match new table name
    ALTER POLICY "org_members_can_manage_documents" ON public.knowledge_sources
      RENAME TO "org_members_can_manage_knowledge_sources";

    -- Rename existing index
    ALTER INDEX IF EXISTS idx_documents_org_id RENAME TO idx_knowledge_sources_org_id;

    -- Step 3: Create LangChain-compatible documents table (vector store)
    CREATE TABLE public.documents (
      id                  BIGSERIAL   PRIMARY KEY,
      content             TEXT        NOT NULL,
      metadata            JSONB       NOT NULL DEFAULT '{}',
      embedding           extensions.vector(1536),
      knowledge_source_id UUID        REFERENCES public.knowledge_sources(id) ON DELETE CASCADE,
      created_at          TIMESTAMPTZ NOT NULL DEFAULT now()
    );

    -- HNSW index for cosine similarity
    CREATE INDEX ON public.documents
      USING hnsw (embedding extensions.vector_cosine_ops);

    -- Index for metadata org_id filtering
    CREATE INDEX idx_documents_metadata_org_id
      ON public.documents USING gin (metadata);

    -- RLS on new documents table
    ALTER TABLE public.documents ENABLE ROW LEVEL SECURITY;

    CREATE POLICY "org_members_can_read_documents"
      ON public.documents
      FOR SELECT
      USING ((metadata->>'org_id')::uuid = public.get_current_org_id());

    -- Service role INSERT policy (edge function uses service key)
    CREATE POLICY "service_role_can_insert_documents"
      ON public.documents
      FOR INSERT
      WITH CHECK (true);

    CREATE POLICY "service_role_can_delete_documents"
      ON public.documents
      FOR DELETE
      USING (true);

    -- Step 4: Create match_documents function (LangChain SupabaseVectorStore signature)
    -- SECURITY DEFINER: bypasses RLS; caller must pass org_id in filter
    CREATE OR REPLACE FUNCTION public.match_documents(
      query_embedding extensions.vector(1536),
      filter          JSONB DEFAULT '{}'
    )
    RETURNS TABLE (
      id         BIGINT,
      content    TEXT,
      metadata   JSONB,
      similarity FLOAT
    )
    LANGUAGE plpgsql
    SECURITY DEFINER
    SET search_path = public, extensions
    AS $$
    BEGIN
      RETURN QUERY
      SELECT
        d.id,
        d.content,
        d.metadata,
        1 - (d.embedding <=> query_embedding) AS similarity
      FROM public.documents d
      WHERE d.metadata @> filter
        AND d.embedding IS NOT NULL
      ORDER BY d.embedding <=> query_embedding ASC;
    END;
    $$;

    Notes:
    - The RLS INSERT/DELETE policies use WITH CHECK (true) / USING (true) because the
      process-embeddings edge function uses the service_role key which bypasses RLS anyway.
      These policies exist only to satisfy the RLS requirement — service_role bypasses them.
    - Do NOT use vector(1536) if extensions.vector is not available — confirm pgvector is
      installed under `extensions` schema (it was enabled in migration 004).
    - Do NOT add a similarity threshold filter inside match_documents — LangChain handles
      thresholding client-side.
  </action>
