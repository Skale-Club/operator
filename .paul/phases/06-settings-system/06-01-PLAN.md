<objective>
## Goal
Criar a infraestrutura de `platform_settings` — tabela no banco, lib de leitura/escrita encriptada e página de admin para gerenciar chaves globais da plataforma (como `GOOGLE_PLACES_API_KEY`).

## Purpose
Configurações globais da plataforma hoje vivem em env vars no Vercel, exigindo redeploy para qualquer mudança. Mover para o banco permite atualização em runtime pelo operador da plataforma, sem toque no Vercel ou no código.

## Output
- Migration `021_platform_settings.sql`
- `src/lib/platform-settings.ts` com `getPlatformSetting` / `setPlatformSetting`
- Página `/settings/platform` acessível apenas ao admin da plataforma
</objective>

    Criar módulo com três funções:
    - `getPlatformSetting(key: string): Promise<string | null>` — usa `createClient()` (autenticado, lê via RLS), descriptografa com `decrypt()`, retorna null se não encontrado
    - `getPlatformSettingHint(key: string): Promise<string | null>` — retorna apenas o hint (sem decrypt, para exibição na UI)
    - `setPlatformSetting(key: string, value: string): Promise<void>` — usa `createServiceRoleClient()` (bypassa RLS), encripta com `encrypt()`, faz upsert na tabela, salva `hint = maskApiKey(value)`

    Importar `encrypt`, `decrypt`, `maskApiKey` de `@/lib/crypto`.
    Importar `createClient` de `@/lib/supabase/server` e `createServiceRoleClient` de `@/lib/supabase/admin`.
    Evitar: expor o plaintext fora da função, usar o admin client para leitura (desnecessário).
