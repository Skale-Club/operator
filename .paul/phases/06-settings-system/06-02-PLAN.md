<objective>
## Goal
Migrar os dois lookups restantes que usam `process.env` incorretamente: (1) `GOOGLE_PLACES_API_KEY` passa a vir de `platform_settings`; (2) `/api/vapi/phone-numbers` passa a ler a chave Vapi da tabela `integrations` da org autenticada — e remover as duas variáveis dos arquivos de env.

## Purpose
Completa o sistema de settings em dois níveis: chaves globais de plataforma no banco (`platform_settings`), chaves por-org no painel de integrações (`integrations`). Nenhum env var de chave de terceiro restará para ser gerenciado fora do admin.

## Output
- `reviews/actions.ts` sem `process.env.GOOGLE_PLACES_API_KEY`
- `/api/vapi/phone-numbers` sem `process.env.VAPI_API_KEY`
- `VAPI_API_KEY` e `GOOGLE_PLACES_API_KEY` removidos de `.env.local` e `.env.local.example`
</objective>

  <name>Task 2: Rota phone-numbers — trocar process.env por integrations por org</name>
  <files>src/app/api/vapi/phone-numbers/route.ts</files>
  <action>
    Refatorar o GET handler:
    1. Importar `createClient` de `@/lib/supabase/server` e `decrypt` de `@/lib/crypto`
    2. Remover `const vapiApiKey = process.env.VAPI_API_KEY`
    3. Buscar a integração Vapi da org autenticada:
       ```ts
       const supabase = await createClient()
       const { data: integration } = await supabase
         .from('integrations')
         .select('encrypted_api_key')
         .eq('provider', 'vapi')
         .eq('is_active', true)
         .single()
       ```
    4. Se `!integration`, retornar `Response.json({ error: 'Vapi integration not configured. Add it in Integrations.' }, { status: 400 })`
    5. `const vapiApiKey = await decrypt(integration.encrypted_api_key)`
    6. Resto da lógica permanece igual (fetch a api.vapi.ai/phone-number).
    
    Nota: `createClient()` já é scoped pela org autenticada via RLS + `get_current_org_id()`, então não é necessário filtrar por org_id explicitamente.
    Evitar: usar service role client aqui (desnecessário e inseguro), mudar o export de runtime.
  </action>
  <verify>npm run build; testar GET /api/vapi/phone-numbers com org que tem integração Vapi → retorna lista de números</verify>
  <done>AC-3 e AC-4 satisfeitas</done>
</task>
