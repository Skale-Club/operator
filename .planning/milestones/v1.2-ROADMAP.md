## Overview

Established the Operator brand (final name landed at end of milestone after pivoting through Leaidear) and shipped an embeddable chat widget for third-party sites, backed by the existing knowledge base and action engine. Visitors converse anonymously via a public per-org token; admins configure appearance and reply to conversations from a new Chat Inbox inside the dashboard.

## Milestone Summary

**Key Decisions:**
- Store widget appearance settings directly on `organizations` instead of a separate widget table — fewer joins, simpler RLS.
- Dedicated token-scoped public GET route for widget boot config with Phase 4 defaults when org values are null.
- Local preview component in admin instead of mounting real `widget.js` — avoids Shadow DOM / script reentrancy inside admin UI.
- Normalize saved widget colors to uppercase `#RRGGBB` so client and server validation enforce the same contract.
- Hydrate admin config *after* mount to keep synchronous `document.currentScript` token capture intact.
- Apply widget primary color via Shadow DOM CSS variable so one update retints bubble, avatars, user bubble, and send button.
- Redis npm package (not `@upstash/redis`) — provider-agnostic URL-based connection.
- `globalThis` HMR guard on Redis client mirrors the Supabase singleton pattern.
- Chat writes use service-role client bypassing RLS; `organization_id` denormalized on `chat_messages`/`conversation_messages` for fast filtering without joins.
- Rename `chat_sessions`/`chat_messages` → `conversations`/`conversation_messages` at Phase 6 — admin-inbox semantics required richer metadata.
- Native fetch + `setInterval` for polling in chat UI (no `@tanstack/react-query` — not installed in this project).
- Renamed platform from Leaidear → **Operator** at end of milestone; canonical origin moved to `https://operator.skale.club`.

**Issues Resolved:**
- Shadow DOM CSS isolation fix in Plan 04-02 resolved all widget styling concerns — Plan 04-03 was verification-only.
- react-resizable-panels v4 API migration — wrapper in `resizable.tsx` translates legacy `direction` prop to new `orientation` prop.
- `vi.hoisted()` fix applied to chat-session test scaffold (Plan 02-02) to stabilize 9/9 GREEN tests.

**Technical Debt (carried forward):**
- No HMAC validation on Vapi webhooks (v1.0 carry-over).
- `send_sms` / `custom_webhook` action types remain stubs (v1.0 carry-over).
- Campaign calls don't auto-appear in Observability (v1.0 carry-over).
- No widget analytics dashboard (deferred to v1.3+).
- No visitor identity capture (optional name/email) before chat (v1.3+).
- No human-agent handoff from widget conversation (v1.3+).
- No widget multi-language support (v1.3+).

**Deferred (v1.3+):**
- Widget analytics (message volume, session count, resolution rate).
- Real-time operator monitoring of live chats.
- Widget A/B testing.
- Email alerts on tool execution failure / latency threshold.
- Client-facing read-only panel (member role dashboard).
