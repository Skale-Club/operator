## Overview

Established the Operator brand (renamed from VoiceOps/Leaidear) and shipped an embeddable chat widget installable on any third-party site via a single `<script>` tag or GTM. The widget is backed by the existing knowledge base (LangChain SupabaseVectorStore) and action engine (executeAction). Admins can configure and deploy the widget per org from the dashboard. Phase 6 added a full admin chat inbox to view, filter, and reply to widget conversations.

## Milestone Summary

### Key Decisions

| Decision | Rationale |
|----------|-----------|
| Widget embed as script tag | Works in any site without framework dependency; GTM compatible |
| Redis for chat session memory | Fast in-session context without hitting Supabase on every message |
| Public org token for widget auth | Visitors don't need accounts; org isolation maintained server-side |
| Widget config on `organizations` table | Avoids extra join/table; natural multi-tenant scope |
| Shadow DOM isolation for widget | Prevents host-site CSS bleed without iframe overhead |
| conversations/conversation_messages naming | More readable table names; renamed via migration 015 |
| Denormalized last_message on conversations | Enables inbox preview without a separate aggregation query |
