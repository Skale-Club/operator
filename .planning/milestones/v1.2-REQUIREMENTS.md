# Operator v1.2 Requirements (Archived)

## Milestone: v1.2 — Operator + Embedded Chatbot

**Status:** ✅ SHIPPED 2026-04-05
**Goal:** Rebrand platform and ship an embeddable chat widget for third-party sites, backed by the existing knowledge base and action engine.

Note: Milestone was originally scoped as "Leaidear + Embedded Chatbot". Final brand landed as **Operator** at end of milestone; canonical origin is `https://operator.skale.club`.

---

## v1.2 Requirements — All Complete

### BRAND — Platform Rename

- [x] **BRAND-01**: Platform UI displays the new brand across all pages, nav, and branding elements — **shipped as "Operator"**
- [x] **BRAND-02**: Page titles, sidebar, login page, and hardcoded references updated

### WIDGET — Embeddable Chat Widget

- [x] **WIDGET-01**: Single `<script>` tag install on any third-party site (no framework dependency)
- [x] **WIDGET-02**: GTM-compatible (async, non-blocking)
- [x] **WIDGET-03**: Floating bubble expands into full chat panel
- [x] **WIDGET-04**: Per-org public token in script tag attributes
- [x] **WIDGET-05**: No visitor login or authentication required

### CHAT — Conversation Engine

- [x] **CHAT-01**: Streamed SSE AI responses in real time
- [x] **CHAT-02**: Responses draw from org's knowledge base (LangChain SupabaseVectorStore)
- [x] **CHAT-03**: AI can call org tools mid-conversation (action engine `executeAction`)
- [x] **CHAT-04**: In-session context maintained via Redis short-term memory
- [x] **CHAT-05**: Conversation history persisted to Supabase (per org, per session)
- [x] **CHAT-06**: Unique anonymous session ID per conversation

### ADMIN — Widget Configuration

- [x] **ADMIN-01**: Admin configures widget appearance per org (name, color, welcome message)
- [x] **ADMIN-02**: Admin page exposes ready-to-copy embed `<script>` tag
- [x] **ADMIN-03**: Live preview of widget with current configuration
- [x] **ADMIN-04**: Admin can regenerate the org's widget public token (invalidates old installs)

### INFRA — Backend Infrastructure

- [x] **INFRA-01**: Redis connection configured and available for chat sessions
- [x] **INFRA-02**: `chat_sessions` / `chat_messages` tables with RLS (renamed to `conversations` / `conversation_messages` in Phase 6)
- [x] **INFRA-03**: Public `/api/chat/[token]` validates token and scopes all queries to org
- [x] **INFRA-04**: Widget asset served from platform's own domain (no external CDN)

### INBOX — Admin Chat Inbox (added mid-milestone as Phase 6)

- [x] **INBOX-01..07**: Admin can view, filter, search, and reply to widget conversations from the dashboard; widget settings relocated under Chat section in sidebar

## Deferred to v1.3+

- Widget analytics dashboard (message volume, session count, resolution rate)
- Visitor identity (optional name/email collection before chat)
- Conversation handoff to human agent
- Widget multi-language support
- Real-time operator monitoring of live chats
- Widget A/B testing

## Deferred to Backlog (carry-over from v1.0)

- Vapi webhook HMAC/secret validation
- `send_sms` action type (Twilio)
- `custom_webhook` action type (configurable URL, method, headers, body)
- Campaign calls auto-appearing in Observability

## Out of Scope

- Visitor login / OAuth for widget users — public token is sufficient
- White-label domain for widget hosting — uses platform domain
- Mobile native SDK — web widget covers mobile browsers
