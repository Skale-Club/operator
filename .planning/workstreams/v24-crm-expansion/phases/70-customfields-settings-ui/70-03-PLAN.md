<task type="auto">
  <name>Task 1: DefinitionModal with per-type conditional form sections</name>
  <files>src/components/settings/custom-fields/definition-modal.tsx</files>
  <action>
    Create 'use client' component at src/components/settings/custom-fields/definition-modal.tsx.

    Props interface:
    ```typescript
    interface DefinitionModalProps {
      open: boolean
      onOpenChange: (open: boolean) => void
      entity: CustomFieldEntity
      definition?: CustomFieldDefinitionRow | null  // null = create mode
    }
    ```

    Mode detection: isEditMode = !!definition?.id.

    Form setup: useForm with zodResolver. In create mode use createDefinitionSchema; in edit mode use updateDefinitionSchema. Default values: for edit, pre-populate from definition prop. For create, defaults: entity=entity prop, required=false, unique_per_org=false, visible_in_list=false, filterable=false.

    Field sections (rendered in order):
    1. Key (text input) — disabled in edit mode. Label: "Field key". Hint below: "Lowercase letters, numbers, underscores. Example: lead_score"
    2. Label (text input) — always editable. Label: "Display label"
    3. Type (Select from all 13 types) — disabled in edit mode with a note "Type cannot be changed after creation"
       Type display labels: text→"Text", long_text→"Long text", number→"Number", integer→"Integer", boolean→"Boolean (Yes/No)", date→"Date", datetime→"Date & time", select→"Single select", multi_select→"Multi-select", url→"URL", email→"Email", phone→"Phone", currency→"Currency"
    4. Group name (text input, optional). Label: "Group". Hint: "Group this field with others under a shared heading"
    5. Help text (Textarea, optional). Label: "Help text". Hint: "Shown below the input as a tooltip or caption"

    Per-type conditional sections (rendered below the base fields, shown only when the selected type matches):
    - When type is 'select' or 'multi_select': Options editor section.
      - options field is an array of { value: string; label: string; color?: string }
      - useFieldArray from react-hook-form for the options array
      - Each option: row with Input for value, Input for label, small color swatch picker (6 preset colors: same PRESET_COLORS as pipeline-settings-client.tsx), Remove button (X icon)
      - DnD for reordering options: wrap in DndContext + SortableContext + useSortable per item
      - "Add option" button below the list
    - When type is 'currency': validation.currency_code text input (3-char uppercase, Label: "Default currency code", placeholder: "USD")
    - When type is 'number' or 'integer': validation.min + validation.max number inputs (Labels: "Min value", "Max value")
    - When type is 'text' or 'long_text': validation.max_length number input (Label: "Max length")

    Toggles section (always shown, in a 2-column grid):
    - required (Switch + Label "Required")
    - unique_per_org (Switch + Label "Unique per org")
    - visible_in_list (Switch + Label "Show in list view")
    - filterable (Switch + Label "Filterable")

    Default value section: simple text Input. Label: "Default value". Hint: "Leave blank for none. For boolean use 'true'/'false'; for select use the option value."

    Form submission:
    - onSubmit: if create mode → call createDefinition(data); if edit mode → call updateDefinition({ id: definition.id, ...data }).
    - On ActionResult.ok === true: toast.success('Field saved'); onOpenChange(false); router.refresh() (to trigger server re-fetch).
    - On ActionResult.ok === false: toast.error(result.error).
    - Use isPending from useTransition for the Submit button loading state.

    Dialog structure: DialogContent with max-w-lg (or xl for complex types). ScrollArea inside for long forms. DialogFooter with Cancel + Save buttons.

    Import useRouter from next/navigation for router.refresh().
