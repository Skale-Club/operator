**`src/components/accounts/account-combobox.tsx`** — add email-domain auto-suggest (per locked decision D-09, ACC-12):

The enhancement: when the user types something containing `@`, extract the email domain and call `getAccounts({ domain: extractedDomain })` in addition to the existing `q`-based search, then merge results (deduplicated by `id`), sorting domain matches first.

Implementation in the debounced `useEffect` that currently calls `getAccounts({ q: inputValue })`:

```typescript
// Inside the debounced setTimeout:
const domainFromEmail = inputValue.includes('@')
  ? inputValue.split('@')[1]?.trim().toLowerCase() ?? ''
  : ''

const [qResult, domainResult] = await Promise.all([
  getAccounts({ q: inputValue, pageSize: 10 }),
  domainFromEmail.length > 0
    ? getAccounts({ domain: domainFromEmail, pageSize: 5 })
    : Promise.resolve({ ok: false as const, error: '' }),
])

const qRows = qResult.ok ? (qResult.data.rows as AccountWithCounts[]) : []
const domainRows = domainResult.ok ? (domainResult.data.rows as AccountWithCounts[]) : []

// Merge: domain matches first, then q matches, deduplicated by id
const seen = new Set<string>()
const merged: AccountWithCounts[] = []
for (const r of [...domainRows, ...qRows]) {
  if (!seen.has(r.id)) { seen.add(r.id); merged.push(r) }
}
setResults(merged.slice(0, 15))
```
