# Plan 85-01: Verify Unified Calls Implementation Completeness

## Objective

Audit the existing `unified_calls` VIEW (migration 063), the `UnifiedCall` TypeScript type block in `database.ts`, and the `getUnifiedCalls`/`getUnifiedCall` server actions in `calls/actions.ts` against the SEED-014 spec. Fix any gaps found. Confirm `npm run build` exits 0 with no type errors.

## Context

All three core deliverables are already implemented (confirmed by research):
- `supabase/migrations/063_unified_calls_view.sql` — VIEW exists with SECURITY INVOKER
- `src/types/database.ts` lines 1779–1806 — `unified_calls` Row type is manually added under `Tables`
- `src/app/(dashboard)/calls/actions.ts` — `getUnifiedCalls(filters)` and `getUnifiedCall(id)` fully implemented

The VIEW columns per SEED-014 spec:
`id, call_type ('ai'|'human'), org_id, external_id, counterpart_number, counterpart_name, contact_id, direction, duration_seconds, status, substatus, recording_url, recording_duration, transcript, notes, cost, assistant_id, routing_mode, started_at, ended_at, created_at`

The ROADMAP success criteria lists `ended_at` as a required column. Verify it is present in both the VIEW SQL and the TypeScript type (the research confirms it is, but the executor must confirm directly).
