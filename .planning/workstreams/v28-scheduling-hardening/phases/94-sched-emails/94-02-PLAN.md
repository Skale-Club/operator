# Plan 94-02 — sendBookingCancellation + integration

## Objective

Add `sendBookingCancellation` to `src/lib/scheduling/emails.ts` and wire both helpers into the action handlers so the booker is notified on confirmation and cancellation.

## Tasks

### Task 1 — sendBookingCancellation (auto)

Add to `src/lib/scheduling/emails.ts`:

- `sendBookingCancellation(params)` accepting bookerEmail, bookerName, hostName, eventTitle, startAt, timezone, rebookUrl
- Subject: `Booking cancelled: {eventTitle} with {hostName}`
- Same dark template style, rebook CTA
- Same fire-and-forget contract

### Task 2 — Wire into action handlers (auto)

In `src/app/(dashboard)/scheduling/_actions/bookings.ts`:

- `createBooking` — after successful insert, fetch host email (from `auth.users` via service-role client) and queue `sendBookingConfirmation` via `void ... .catch(() => {})`. Use `${SITE_URL}/book/cancel/{bookingId}?token={cancelToken}` for the cancel URL.
- `cancelBookingByToken` — when update succeeds, re-fetch booking + event type + host email, queue `sendBookingCancellation`. Use `${SITE_URL}/book/{profileSlug}/{eventSlug}` for rebook.
- `cancelBooking` (dashboard) — same pattern as cancelBookingByToken.

For the host name we use the `auth.users.email` since `scheduling_profiles` does not store a display name. Use `process.env.NEXT_PUBLIC_SITE_URL ?? 'https://xphere.skale.club'` as base.

## Verification

- Both functions exported from `src/lib/scheduling/emails.ts`
- All three action paths (createBooking, cancelBookingByToken, cancelBooking) invoke the email helpers
- Booking action returns success even if email send fails (verified by void+catch wrapper)
- `npm run build` exits 0
