# Plan 58-01: Migration 058 + database.ts types

## Goal

Ship migration `058_twilio_phone_numbers.sql` end-to-end: DDL, RLS, indexes, backfill, and matching TypeScript types in `src/types/database.ts`. After this plan lands, `npm run build` passes and the schema is ready for Phase 59 server actions to query.

## Tasks

### Task 1: Write the migration

**File:** `supabase/migrations/058_twilio_phone_numbers.sql`

Single-transaction migration following the canonical pattern from `054_evolution_instances.sql` and `002_action_engine.sql` (parent `integrations` table).

Use `organization_id` (matches the parent `integrations` table column convention — older tables in this codebase use `organization_id`, newer ones use `org_id`; matching the parent table makes joins natural).

Reuse the shared `public.update_updated_at()` trigger function (defined in migration 001). Do NOT create a new touch function.

Structure:

```sql
-- Migration 058: Twilio multi-number support (v2.3)
-- Promotes "from_number" from a JSONB string to a first-class per-org entity.
-- Each org can register multiple Twilio numbers with capabilities, friendly
-- labels, and a default flag for outbound routing. Backfills the existing
-- `integrations.config.from_number` value as a default number per org.

BEGIN;

CREATE TABLE IF NOT EXISTS public.twilio_phone_numbers (
  id                    UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id       UUID         NOT NULL REFERENCES public.organizations(id) ON DELETE CASCADE,
  e164                  TEXT         NOT NULL,
  phone_sid             TEXT,
  friendly_name         TEXT         NOT NULL,
  capability_sms        BOOLEAN      NOT NULL DEFAULT false,
  capability_mms        BOOLEAN      NOT NULL DEFAULT false,
  capability_voice      BOOLEAN      NOT NULL DEFAULT false,
  default_routing_mode  TEXT         CHECK (default_routing_mode IS NULL OR default_routing_mode IN ('browser', 'sip', 'forward')),
  forward_to_number     TEXT,
  is_default            BOOLEAN      NOT NULL DEFAULT false,
  is_active             BOOLEAN      NOT NULL DEFAULT true,
  notes                 TEXT,
  created_at            TIMESTAMPTZ  NOT NULL DEFAULT now(),
  updated_at            TIMESTAMPTZ  NOT NULL DEFAULT now(),
  UNIQUE (organization_id, e164)
);

-- Indexes
CREATE INDEX IF NOT EXISTS idx_twilio_phone_numbers_org
  ON public.twilio_phone_numbers (organization_id);

CREATE INDEX IF NOT EXISTS idx_twilio_phone_numbers_e164_active
  ON public.twilio_phone_numbers (e164)
  WHERE is_active = true;

-- Exactly zero or one default per org (DB-enforced; race-safe)
CREATE UNIQUE INDEX IF NOT EXISTS twilio_phone_numbers_one_default_per_org
  ON public.twilio_phone_numbers (organization_id)
  WHERE is_default = true;

-- updated_at trigger (reuses shared function from migration 001)
DROP TRIGGER IF EXISTS trg_twilio_phone_numbers_updated_at ON public.twilio_phone_numbers;
CREATE TRIGGER trg_twilio_phone_numbers_updated_at
  BEFORE UPDATE ON public.twilio_phone_numbers
  FOR EACH ROW EXECUTE FUNCTION public.update_updated_at();

-- RLS: standard per-org isolation
ALTER TABLE public.twilio_phone_numbers ENABLE ROW LEVEL SECURITY;

DROP POLICY IF EXISTS twilio_phone_numbers_org_isolation ON public.twilio_phone_numbers;
CREATE POLICY twilio_phone_numbers_org_isolation
  ON public.twilio_phone_numbers
  FOR ALL
  TO authenticated
  USING (organization_id = (SELECT public.get_current_org_id()))
  WITH CHECK (organization_id = (SELECT public.get_current_org_id()));

-- Backfill from integrations.config.from_number
-- One row per active Twilio integration with a non-empty from_number.
-- Uses ON CONFLICT to make the migration safely re-runnable.
INSERT INTO public.twilio_phone_numbers (
  organization_id,
  e164,
  friendly_name,
  capability_sms,
  capability_voice,
  capability_mms,
  is_default,
  is_active
)
SELECT
  organization_id,
  config->>'from_number'                AS e164,
  COALESCE(config->>'from_number', '')  AS friendly_name,
  true,
  true,
  false,
  true,
  true
FROM public.integrations
WHERE provider = 'twilio'
  AND is_active = true
  AND config ? 'from_number'
  AND config->>'from_number' IS NOT NULL
  AND config->>'from_number' <> ''
ON CONFLICT (organization_id, e164) DO NOTHING;

COMMIT;
```

**Verify the file compiles by running** `npx supabase db diff` (read-only) or visually reviewing the SQL.

### Task 2: Update `src/types/database.ts`

Add the `twilio_phone_numbers` entry inside `public.Tables`, mirroring the style of `evolution_instances` (which is a few lines above the insertion point). Use `organization_id` (not `org_id`).

Shape:

```ts
twilio_phone_numbers: {
  Row: {
    id: string
    organization_id: string
    e164: string
    phone_sid: string | null
    friendly_name: string
    capability_sms: boolean
    capability_mms: boolean
    capability_voice: boolean
    default_routing_mode: 'browser' | 'sip' | 'forward' | null
    forward_to_number: string | null
    is_default: boolean
    is_active: boolean
    notes: string | null
    created_at: string
    updated_at: string
  }
  Insert: {
    id?: string
    organization_id: string
    e164: string
    phone_sid?: string | null
    friendly_name: string
    capability_sms?: boolean
    capability_mms?: boolean
    capability_voice?: boolean
    default_routing_mode?: 'browser' | 'sip' | 'forward' | null
    forward_to_number?: string | null
    is_default?: boolean
    is_active?: boolean
    notes?: string | null
    created_at?: string
    updated_at?: string
  }

  ]
```
